Client/Server using mutual TLSv1.3 in Java

Intro

In this tutorial, I will show you how to create a secure channel between two java applications communicating through the new TLSv1.3.
The benefit of using TLSv1.3 is:

  • Authentication and Speed: Quickly authenticate the server and client identity.
  • Confidentiality: Encrypt data transmitted between the server and client.
  • Data Integrity: Protect against data modification.
Since I am not a security expert I will not delve into the details of how TLS(Transport Layer Security) works, but I can briefly explain what you need to know to create your application. Both the server and client need to generate a keystore, which is a file that holds the application private key and public key(certificate) and also a truststore which holds trusted certificates.

If you are interested to learn more visit: Java Secure Socket Extension (JSSE) Reference Guide

How it works:

This is how it is going to work. Unlike RSA that requires 4 steps to finish the handshake, TLSv1.3 uses diffie-Hellman Ephemeral(DHE) to cut the time in half. Below is a figure showing the full TLS handshake: In Short:

  1. The ClientHello will contain shared random key, and cryptographic parameters.
  2. The ServerHello agrees to DH params, checks if client needs to authenticate, then combines ClientHello with ServerHello to generate a master key.
  3. The client receives the ServerHello generates the master key and responds with its authentication message(certificate) if requested.
  4. Now both server and client have a master key to encrypt data application.
for more information visit: rfc8446 or watch this video tutorial:

Let's get our hands dirty:

First, we need to generate a keystore: To keep this tutorial short I will use the same keystore as a truststore for both client and server.
keytool -genkeypair -alias mykey -keystore mykeystore.p12 -storetype pkcs12 -storepass secretpass -validity 3650 -keysize 2048 -keyalg RSA
To learn more about keytool command and options visit: Keytool keystore implementation

Second, we create two java projects one called ServerTLS and the other one ClientTLS using your preferred IDE. I am using Intellij. We move the keystore to both ServerTLS and ClientTLS. and load it as a keystore and truststore using System.setProperty.
